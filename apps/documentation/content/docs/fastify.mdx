## When to use

Use the Fastify handler when your app is built with **Fastify**. The handler is a Fastify plugin that registers all routes under the base path (`/flags/*`).

## Installation

```bash
bun add @better-flag/core @better-flag/adapter-memory @better-flag/handler-fastify
```

## Setup

1. Create the engine (see [Installation](/docs/installation)).
2. Register the handler with `app.register()`. Fastify plugins are async.

```ts title="app.ts"
import Fastify from "fastify";
import { flagEngine } from "@/lib/flags";
import { createFastifyHandler } from "@better-flag/handler-fastify";

const app = Fastify();

await app.register(createFastifyHandler(flagEngine, {
  basePath: "/flags",
  getContext: (req) => {
    const user = (req as any).user;
    return user ? { userId: user.id, attributes: {} } : null;
  },
  requireAuth: false,
}));

await flagEngine.init();
await app.listen({ port: 3000 });
```

**basePath**: The handler expects requests at `{basePath}`, `{basePath}/:key`, `{basePath}/evaluate`, etc. With `basePath: "/flags"`, the full paths are `/flags`, `/flags/:key`, `/flags/evaluate`.

## getContext

The callback receives `req` (Fastify request). The user is usually set by an auth plugin or decorator that runs before the flags handler.

```ts
getContext: (req) => {
  const user = (req as any).user;
  if (!user) return null;
  return {
    userId: user.id,
    attributes: { plan: user.plan },
  };
},
```

**Note**: Use Fastify decorators (e.g. `fastify.decorateRequest('user', null)`) to type the user. Ensure your auth plugin runs before the flags handler.

## Options

| Option | Type | Description |
|--------|------|-------------|
| `basePath` | `string` | Path prefix (e.g. `/flags`) |
| `getContext` | `(req: FastifyRequest) => EvaluationContext \| null` | Extract context from request. Return `null` when unauthenticated. |
| `requireAuth` | `boolean` | If `true`, return 401 when `getContext` returns `null` for evaluate routes |

## Routes exposed

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `{basePath}` | List all flags |
| `POST` | `{basePath}` | Create flag |
| `GET` | `{basePath}/:key` | Get flag by key |
| `PATCH` | `{basePath}/:key` | Update flag |
| `DELETE` | `{basePath}/:key` | Delete flag |
| `POST` | `{basePath}/evaluate` | Evaluate flag(s) |

See [API Reference](/docs/api) for request/response formats.
