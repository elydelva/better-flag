## Package layers

```
types/         тЖТ Shared domain types (Flag, EvaluationContext, EvaluationResult, etc.). Used by core, client, adapters.
core/          тЖТ Pure engine. Interfaces. Evaluation pipeline. Zero external imports.
adapters/      тЖТ Concrete implementations of FlagAdapter. Memory, file, Redis, Drizzle.
handlers/      тЖТ Optional HTTP bridges. Next.js, Hono, Express, Fastify. Take FlagEngine, expose routes.
client/        тЖТ Type-safe HTTP client. createBetterFlagClient.
plugins/       тЖТ Extensions: overrides, rate-limit, rules, audit. Evaluators, schema, services.
cli/           тЖТ Schema generation and migrations. bf generate, bf migrate.
```

## Types package

`@better-flag/types` provides shared domain types:

- `Flag`, `FlagType`, `FlagValue`, `FlagVariant`
- `RolloutConfig`, `ScheduleConfig`
- `EvaluationContext`, `EvaluationResult`, `EvalReason`
- `Condition`, `ConditionOperator`, `TargetingRule`

Import from `@better-flag/types` or `@better-flag/core` (re-exports).

## Extensibility

The engine is designed for plugins that add:

- **Evaluators** тАФ Priority-ordered evaluation (overrides, rules, rollout, default)
- **Routes** тАФ Merged with core routes (if needed)
- **Services** тАФ Attached via `createServices` (e.g. `engine.getPlugin("overrides")`)
- **Schema** тАФ Tables via `schemaContribution` for Drizzle/Prisma

## FlagAdapter interface

The core defines this interface. The adapter implements it.

| Area | Methods |
|------|---------|
| **flags** | `get`, `getAll`, `set`, `delete` |
| **extensions** | Optional (e.g. `getPluginData` for Redis) |
| **subscribe** | Optional change notifications |
| **onReady** | Initialize before use |

## Design principles

1. **Core pure** тАФ No framework, ORM, or runtime in core. Testable without DB or HTTP.
2. **Interfaces-first** тАФ Core defines interfaces; adapters implement them.
3. **Evaluators = pipeline** тАФ Plugin evaluators run by priority; first match wins.
4. **Zero vendor** тАФ No external SaaS required.
5. **Type-safety end-to-end** тАФ Flag types flow from config to client.
6. **Schema language** тАФ Plugins contribute schema; CLI generates Drizzle/Prisma.
